# 커밋 메시지 작성 가이드
# ----------------------
# 제목은 아래 형식 중 하나로 작성하세요:
#
# - feat: 새로운 기능 추가
# - fix: 버그 수정
# - test: 테스트 코드 작성/수정
# - refactor: 코드 리팩토링
# - style: 코드 스타일 변경 (로직 변경 없음)
# - chore: 기타 작업 (예: 빌드 설정 변경)
#
# 본문 작성은 선택 사항입니다.
# 본문은 '무엇을', '왜' 변경했는지 간단히 설명하세요.
#
# 예:
# feat: 사용자 로그인 기능 추가
#
# - JWT 토큰 발행 기능 구현
# - 이메일 중복 확인 로직 추가
#
